# Dependency Injection với GetIt & Injectable

Trong app lớn, có rất nhiều Cubit và Repo. Ví dụ `HomeCubit` cần dùng `AuthService`, nếu chúng ta tự truyền tay:

```dart
final authService = AuthService();
final homeCubit = HomeCubit(authService);
```

Việc này sẽ gây rối rắm. Thay vào đó, chúng ta dùng **Dependency Injection (DI)** thông qua 2 package: `get_it` và `injectable`.

## 1. Bản chất

- **GetIt**: Giống như 1 chiếc hộp thần kỳ. Bạn ném class vào đó, sau đó đi bất cứ đâu trong code gọi `getIt<TênClass>()` là chiếc hộp sẽ lấy đúng class đó ra cho bạn.
- **Injectable**: Là code generator (máy tự viết code). Thay vì tự tay ném từng class vào rương GetIt, ta chỉ cần gắn các annotation (`@injectable`, `@lazySingleton`), nó sẽ tự viết code bỏ vào hộp.

## 2. Các loại Annotation thường dùng

### `@lazySingleton`
Chỉ tạo **1 instance duy nhất** trong toàn bộ vòng đời của app. Phù hợp cho `AuthService`, `HomeCubit` (nếu muốn đếm số lượng thông báo được giữ liên tục ở mọi màn).

```dart
@lazySingleton
class AuthService {
  User? currentUser;
}
```

Khi ở trang khác:
```dart
final user = getIt<AuthService>().currentUser; // Vẫn là user đó
```

### `@injectable`
Mỗi lần gọi `getIt<TênClass>()` nó sẽ tạo ra **1 instance mới hoàn toàn**. Phù hợp với các form như `ScheduleFormCubit` (nghỉ nhập form, vào lại thì form phải trống như mới).

```dart
@injectable
class NotificationCubit extends BaseCubit<NotificationState> {
  NotificationCubit() : super(...);
}
```

### Đăng ký interface với `@Injectable(as: ...)`
Thường áp dụng cho Repository pattern. Ta có 1 class abstract `NotificationRepo` và 1 class thực hiện nó `NotificationRepoImpl`.

```dart
@Injectable(as: NotificationRepo)
class NotificationRepoImpl implements NotificationRepo {
  // Logic kết nối API...
}
```

Sau này Cubit chỉ cần gọi `NotificationRepo` mà không cần quan tâm RepImpl là gì:
```dart
@injectable
class NotificationCubit {
  final NotificationRepo _repo; // GetIt tự chắp nối!

  NotificationCubit(this._repo);
}
```

## 3. Quên chạy `build_runner` là lỗi!
Sau khi bạn thêm `@injectable` hoặc sửa constructor, bạn sẽ thấy code báo lỗi (không tìm thấy Dependency). Bạn **bắt buộc** phải chạy lệnh sau ở terminal (đứng ở thư mục `frontend`):

```bash
dart run build_runner build --delete-conflicting-outputs
```

Lệnh này sẽ tự viết ra file `di_config.config.dart`. **Tuyệt đối không bao giờ sửa tay file tự động sinh này.**
