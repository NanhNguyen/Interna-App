# 9. State Management Flow (Luồng Quản Lý Trạng Thái)

Với sự phức tạp của ShiftFlow, đội ngũ phát triển tin dùng **BloC/Cubit** (Business Logic Component). Nhiệm vụ của nó là tách riêng **Phần Hiển Thị Của App (UI)** và **Logic Nghiệp vụ (Data/API)**.

## 🌟 1. Vấn đề của Flutter nguyên thủy nếu bỏ qua Cubit
Cách dùng `setState` thông thường bắt lỗi:
1. `setState` bắt Flutter vẽ lại (rebuild) toàn bộ một màn hình, dù đôi khi bạn chỉ muốn update 1 cái màu chữ. Quá giật!
2. Call API trộn lẫn ngay trên file Giao diện (`.dart`). Lúc sau đọc lại code như "Rác code". Rất khó bảo trì!

Cubit giải quyết điều này: UI chỉ lắng nghe (Listen) trạng thái do Cubit nhả (Emit) ra, rồi báo lên các sự kiện từ user bấm nút ở chiều ngược lại.

## 🌟 2. Cubit là gì? State là gì?
1. **State**: Là Trạng thái. Ví dụ quá trình Đăng nhập sẽ sinh ra 3 state: Đang Load (`loading`), Thành Công (`success`), hoặc Thất Bại (`error`).
2. **Cubit**: Là cái "Nhà Máy" tiếp nhận yêu cầu từ người dùng (như bấm nút Nộp Đơn Xin Nghỉ), tiếp tục nói chuyện với API, sau đó sản xuất (Emit) ra các State chuyển về lại UI.

## 🌟 3. Quy trình chuẩn từ Cú Click chuột -> Đổi màn hình

### Bước 1: User hành động
User bấm nút "Cập nhật Mật Khẩu" trên `ProfilePage`. UI sẽ không tự làm gì cả, mà nó sẽ gửi tin nhắn đến Cubit qua hàm gọi:
```dart
onPressed: () {
  // Gửi lệnh cho Nhà Máy Cubit làm việc!
  context.read<ProfileCubit>().changePassword('passMoi');
}
```

### Bước 2: Cubit xử lý Data và nôn ra State
Trong file `profile_cubit.dart`, hàm `changePassword` được bắt đầu:
```dart
Future<void> changePassword(String newPass) async {
  // Báo tin về UI: Màn hình hiển thị "Loading vòng xoay" đi
  emit(state.copyWith(status: BaseStatus.loading)); 
  
  try {
    // Nhờ ông Coder bên dưới Data Layer gọi API
    await userRepository.updatePass(newPass);

    // Báo tin về UI: Thành công rồi, tắt vòng xoay và báo xanh lá nha
    emit(state.copyWith(status: BaseStatus.success)); 
  } catch(e) {
    // Báo tin về UI: Báo đỏ lỗi mạng/sai pass
    emit(state.copyWith(status: BaseStatus.error, error: 'Sai mật khẩu cũ')); 
  }
}
```

### Bước 3: UI tự Đổi theo (BlocListener / Builder)
- Dùng `BlocBuilder` khi bạn muốn 1 bộ phận màn hình update Realtime theo State (Như hiển thị Avater, Loading text...).
- Dùng `BlocListener` khi hành động xong bạn muốn làm việc tĩnh như Nhảy qua Trang Mới (Route) hay Hiển thị Thông Báo Pop-up (SnackBar).

```dart
// Lắng Nghe - Không vẽ UI, chỉ tạo tác vụ chìm
BlocListener<ProfileCubit, ProfileState>(
  listener: (context, state) {
    if (state.status == BaseStatus.success) {
      showDialogThanhCong();
      router.push(...); // Chuyển trang
    }
  },
  child: // Phần Giao diện (Màn hình sẽ KHÔNG BỊ Rebuild nhờ tách biệt code)
)
```

## 💡 Mẹo Tối Ưu Tốc Độ Web Flutter (Tránh Rebuild Vô Lập)
Bên cạnh State của phần tử, đừng quên chặn **Rebuild** thừa ở UI bằng cái `buildWhen`:
```dart
BlocBuilder<UserCubit, UserState>(
  // LUẬT VÀNG: Chỉ vẽ lại cái ô Tên Người Dùng nếu name bị thay đổi.
  buildWhen: (previousState, currentState) => previousState.name != currentState.name,
  builder: (context, state) {
     return Text(state.name);
  }
)
```
