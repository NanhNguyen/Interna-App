# Freeze (Model) - Gọi API nhàn hơn

Trong Dart, việc viết 1 class để hứng json khá vất vả vì bạn phải tự viết tay `fromJson`, `toJson`, `copyWith`, hay `hashCode`. `freezed` và `json_serializable` sinh ra để khắc phục điều này.

## 1. Class Data "Đông lạnh" (Immutable)
Trạng thái/model khi đã lấy từ API không nên bị sửa lung tung. `Freezed` chỉ tạo class kiểu đọc-chỉ (`final`), an toàn và sinh tự động.

## 2. Cách tạo 1 Model chuẩn

```dart
// Ví dụ tạo 1 UserModel ở file data/model/user_model.dart:

import 'package:freezed_annotation/freezed_annotation.dart';

// Phải có 2 dòng này! (phần tên file thay vì user_model hãy ghi đúng tên file mình tạo ra)
part 'user_model.freezed.dart';
part 'user_model.g.dart';

@freezed
class UserModel with _$UserModel {
  // Annotation này giúp đổi key json "user_id" thành property "userId"
  const factory UserModel({
    @JsonKey(name: '_id') required String id,
    required String name,
    required String email,
    required String role,
  }) = _UserModel;

  // Cực kỳ quan trọng, sinh hàm Parse tự động
  factory UserModel.fromJson(Map<String, dynamic> json) => 
      _$UserModelFromJson(json);
}
```

dart run build_runner build --delete-conflicting-outputs

Khá mất thời gian đúng không? Khi bạn lưu file nó sẽ báo đỏ khè vì không tìm được phần `.g.dart` và `.freezed.dart`. Chạy xong câu lệnh trên là hết!

## 4. Cách sử dụng

### Decode JSON (Từ Server về)
Ở file Repo của bạn (khi gọi `dio.get`):
```dart
final response = await dio.get('/user');
// Decode List:
final List users = (response.data as List).map((e) => UserModel.fromJson(e)).toList();

// Decode 1 Obj:
final user = UserModel.fromJson(response.data);
```

### Encode lại lên server
```dart
final jsonData = user.toJson(); 
await dio.post('/user', data: jsonData);
```

### Cập nhật (Copy) Model
Vì Freezed class là read-only, nên nếu bạn cần cập nhật tên user?
```dart
// Tạo 1 bản ghi MỚI dựa trên thông tin bản cũ => tránh chèn đè 
final newNameUser = user.copyWith(name: "Nam Anh");
```
