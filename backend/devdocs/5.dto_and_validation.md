# 5. DTO & Validation trong NestJS

Trong dự án ShiftFlow, chúng ta sử dụng **DTO (Data Transfer Object)** kết hợp với `class-validator` và `class-transformer` để kiểm tra (validate) và làm sạch dữ liệu đầu vào. Điều này giúp ngăn chặn dữ liệu bẩn lọt vào database.

## 🌟 1. DTO là gì?
DTO đơn giản là một class mô tả hình dạng dữ liệu (payload) mà client gửi lên server (trong body, query, hay param).

Ví dụ: Khi Manager duyệt một Request, họ sẽ gửi lên `status` (APPROVED hoặc REJECTED). DTO sẽ định nghĩa điều này.

## 🌟 2. Cấu trúc một File DTO điển hình
Thư mục chứa: Thường nằm trong thư mục của module, ví dụ: `src/requests/dto/update-request.dto.ts`.

```typescript
import { IsEnum, IsNotEmpty, IsOptional, IsString } from 'class-validator';
import { RequestStatus } from '../enums/request-status.enum';

export class UpdateRequestStatusDto {
  // Bắt buộc phải có, không được rỗng
  @IsNotEmpty({ message: 'Trạng thái không được để trống' })
  // Phải thuộc 1 trong các giá trị của Enum RequestStatus
  @IsEnum(RequestStatus, { message: 'Trạng thái không hợp lệ' })
  status: RequestStatus;

  // Trường này là tùy chọn (không có cũng không sao)
  @IsOptional()
  @IsString({ message: 'Lý do từ chối phải là chuỗi văn bản' })
  rejectionReason?: string;
}
```

## 🌟 3. Cách Controller sử dụng DTO
Thay vì lấy dữ liệu thủ công từ `@Body() body: any`, chúng ta sẽ gắn trực tiếp class DTO vào.

```typescript
import { Body, Controller, Patch, Param } from '@nestjs/common';
import { UpdateRequestStatusDto } from './dto/update-request.dto';

@Controller('requests')
export class RequestsController {
  
  @Patch(':id/status')
  async updateStatus(
    @Param('id') id: string,
    @Body() updateDto: UpdateRequestStatusDto // Dữ liệu tự động được validate dựa theo DTO
  ) {
    // Nếu dữ liệu gửi lên sai (vd: status = 'UNKNOWN_STATUS')
    // NestJS sẽ tự động chặn lại và trả về lỗi 400 Bad Request cho Frontend
    // Code bên trong hàm này chỉ chạy khi dữ liệu đã chuẩn 100%!
    
    return this.requestsService.updateStatus(id, updateDto);
  }
}
```

## 🌟 4. Các Decorator thường dùng
- `@IsString()`, `@IsNumber()`, `@IsBoolean()`: Kiểm tra kiểu dữ liệu cơ bản.
- `@IsNotEmpty()`: Yêu cầu trường này phải có giá trị.
- `@IsOptional()`: Nếu không gửi lên thì bỏ qua kiểm tra các luật phía dưới.
- `@IsEmail()`: Phải là đúng định dạng email.
- `@Min(giá_trị)`, `@Max(giá_trị)`: Limit giá trị của Number.
- `@IsEnum(TênEnum)`: Giá trị gửi lên phải thuộc chuẩn Enum khai báo sẵn.

## 💡 Mẹo nhỏ cho Dev mới
* Luôn khai báo `message` tùy chỉnh trong class-validator (như ví dụ trên) bằng tiếng Việt để Frontend có thể hiển thị câu báo lỗi thân thiện nhất cho User!
