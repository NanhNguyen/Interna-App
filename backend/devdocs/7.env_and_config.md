# 7. `.env` & Configuration

Dự án ShiftFlow quản lý các thông tin nhạy cảm (Tài khoản DB, Khóa bảo mật JWT, Port...) bằng tệp `.env`. Tệp này **tuyệt đối không được push lên Github**.

## 🌟 1. Cấu trúc file `.env` chuẩn
Tạo file `.env` ở thư mục root của `backend/` với nội dung cơ bản sau:
```env
# Cổng chạy ứng dụng Local
PORT=3000

# Đường dẫn DB
MONGODB_URI=mongodb+srv://<username>:<password>@cluster.mongodb.net/shiftflow_db

# Mã bí mật để ký token
JWT_SECRET=shiftflow_secret_key_123_@!!!
JWT_EXPIRES_IN=7d
```

## 🌟 2. Sử dụng `@nestjs/config`
Thay vì gọi `process.env.MONGODB_URI` lộn xộn khắp nơi (cách cũ của NodeJS), ShiftFlow cài đặt `ConfigModule` ngay tại `app.module.ts`:

```typescript
import { Module } from '@nestjs/common';
import { ConfigModule, ConfigService } from '@nestjs/config';
import { MongooseModule } from '@nestjs/mongoose';

@Module({
  imports: [
    // 1. Khởi tạo Config Module
    ConfigModule.forRoot({
      isGlobal: true, // Biến env dùng ở đâu cũng được
    }),
    
    // 2. Chích biến env vào Mongo Connect
    MongooseModule.forRootAsync({
      imports: [ConfigModule],
      inject: [ConfigService],
      useFactory: async (configService: ConfigService) => ({
        uri: configService.get<string>('MONGODB_URI'),
      }),
    }),
  ],
})
export class AppModule {}
```

## 🌟 3. Gọi biến Env bên trong Service
Trong bất kỳ Service nào, bạn cũng có thể gọi ra biến `.env`:

```typescript
import { Injectable } from '@nestjs/common';
import { ConfigService } from '@nestjs/config';

@Injectable()
export class AuthService {
  constructor(private configService: ConfigService) {}

  printSecret() {
    const jwtKey = this.configService.get<string>('JWT_SECRET');
    console.log('Secret key là: ', jwtKey);
  }
}
```

## 💡 Mẹo nhỏ cho Dev mới 
* Ở các nền tảng deploy thật (Render, Vercel), chúng ta sẽ không tải lên file `.env` mà sẽ phải thiết lập thủ công trong phần cài đặt **"Environment Variables"** của Dashboard trang chủ nền tảng đó.
* Nên có một tệp `env.example` liệt kê danh sách tên các biến (không chứa mật khẩu thật) push lên git. Mục đích để khi ai đó clone dự án về, họ biết là phải tạo `.env` có những trường (fields) gì.
